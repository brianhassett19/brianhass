ISLR_orange <- "#CC5318"

# Return vector of the names of numeric columns in a data frame:
numeric_cols <- function(df) {
  is_numeric_cols <- map_lgl(df, is.numeric)
  return(names(df)[is_numeric_cols])
}

# Plot histograms and return a list of them:
plot_histogram <- function(df, var) {
  g <- ggplot(data = df, aes(x = .data[[var]])) + # Ref for passing var names to functions for plots: https://stackoverflow.com/a/55524126
    geom_histogram(bins = 30,
                   color = "white")
  return(g)
}

# Plot boxplots and return a list of them:
plot_boxplot <- function(df, var1, var2) {
  g <- ggplot(data = df, aes(x = .data[[var1]], y = .data[[var2]])) +
     geom_boxplot() 
  return(g)
}

Calculate the euclidean distances and correlations (\(r_ij\)):

X.scale <- t(scale(t(USArrests))) # scale observations, not features (https://altaf-ali.github.io/ISLR/chapter10/solutions.html)

euc <- dist(X.scale)
corr <- as.dist(1 - cor(t(X.scale))) # 1 - r_ij

# Plot the values
data.frame(x = as.numeric(euc^2), y = as.numeric(corr)) %>%

  sum()

The variance explained by the \(m\)th principal component (the top half of (10.8)) is the sum of the squared principal component scores. For the first principal component (\(m=1\)) the variance explained is:

(pr.out$x[ , 1])^2 %>% sum()
[1] 121.5318

The total variance in the data (bottom half of (10.8)) is the sum of the squares of the data values (remember to scale the values before summing):

var.total <- scale(USArrests)^2 %>% sum()
var.total
[1] 196

Then the proportion of variance explained by the first principal component is found dividing the two (as in (10.8)):

((pr.out$x[ , 1])^2 %>% sum() / var.total) %>% round(2)
[1] 0.62

Repeat for all four principal components:

pve.10.8 <- map_dbl(1:4, function(m) (pr.out$x[ , m])^2 %>% sum() / var.total)
pve.10.8 %>% round(2)
[1] 0.62 0.25 0.09 0.04

Exercise 10

  (a)

Generate a simulated data set with 20 observations in each of three classes (i.e. 60 observations total), and 50 variables:

set.seed(123)
m <- 60
n <- 50
sim <- matrix(nrow = m, ncol = n)
for(i in 1:n) {
  mu <- runif(1, 0, 5)
  sd <- runif(1, 1, 5)
  sim[1:(m/3), i] <- rnorm(m/3, mu, sd)
  sim[(m/3 + 1):(2*m/3), i] <- rnorm(m/3, mu + 0.5*mu, sd) # mean shift
  sim[(2*m/3 + 1):m, i] <- rnorm(m/3, mu - 0.5*mu, sd) # mean shift
}
labels <- c(rep(1, 20), rep(2, 20), rep(3, 20))

\(K\)-means clustering with \(k = 3\) separated the observations into three clusters of unequal size (15, 23, 22). Of the observations with a true label of 1, \(k\)-means assigned 11 to cluster 1, 4 to cluster 2 and 5 to cluster 3. Of the observations with a true label of 2, \(k\)-means assigned 1 to cluster 1, 19 to cluster 2 and 0 to cluster 3. Of the observations with a true label of 3, \(k\)-means assigned 3 to cluster 1, 0 to cluster 2 and 17 to cluster 3.

Overall, \(k\)-means performed well on the observations with label 2 and 3, but assigned almost half of the observations labeled 1 to other clusters.

This time \(k\)-means has 4 clusters to assign observations with only 3 different labels. Again, \(k\)-means splits up the observations with a true label of 1, with 2, 6, 10 and 2 observations in the four clusters. It correctly groups together 19 of the observations with label 2, with one observation assigned to another cluster. The grouping is poorer on the observations with label 3; 12 are assigned to cluster 1, but 8 are put in different clusters, 2 in cluster 3 and 6 in cluster 2. There is only one instance of crossover between true labels 2 and 3, which is the single observation with label 2 that was assigned to cluster 3.
